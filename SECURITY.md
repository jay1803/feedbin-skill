# Security Policy

## Supported Versions

- `0.2.x`: supported

## Reporting a Vulnerability

Report issues privately to the maintainer before public disclosure.

When reporting, include:

- affected version (`VERSION` file)
- reproduction steps
- expected vs actual behavior
- impact assessment (credential exposure, data loss, etc.)

## Security Notes

- Do not commit `FEEDBIN_EMAIL`, `FEEDBIN_PASSWORD`, or `.env` files.
- Prefer least-privilege account access for Feedbin automation.
